## 4. HTTP 接口字段参考

### 4.1 新浪财经 API

#### 实时行情接口

```
GET http://hq.sinajs.cn/list=sh600000,sz000001
```

**返回字段说明：**

| 字段序号 | 字段名称 | 说明 |
|---------|---------|------|
| 0 | 名称 | 股票/指数名称 |
| 1 | 今日开盘价 | 当日开盘价 |
| 2 | 昨日收盘价 | 上一交易日收盘价 |
| 3 | 当前价格 | 最新成交价 |
| 4 | 今日最高价 | 当日最高成交价 |
| 5 | 今日最低价 | 当日最低成交价 |
| 6 | 竞买价 | 买一价 |
| 7 | 竞卖价 | 卖一价 |
| 8 | 成交量 | 当日累计成交量（股） |
| 9 | 成交金额 | 当日累计成交额（元） |
| 10-14 | 买一到买五价 | 五档买入价格 |
| 15-19 | 买一到买五量 | 五档买入数量 |
| 20-24 | 卖一到卖五价 | 五档卖出价格 |
| 25-29 | 卖一到卖五量 | 五档卖出数量 |
| 30 | 日期 | 最近逐笔成交日期 YYYY-MM-DD |
| 31 | 时间 | HH:mm:ss |
| 32 | 涨跌 | 当前价 - 昨收 |
| 33 | 涨跌幅 | (当前价 - 昨收) / 昨收 * 100% |
| 38 | 换手率 | 当日换手率 |
| 39 | 市盈率 | PE值 |

**股票代码规则：** `sh` 上海（sh600000）、`sz` 深圳（sz000001）、`hk` 港股（hk00700）、`gb` 美股（gb$aapl）

#### 历史K线接口

```
GET https://money.finance.sina.com.cn/quotes_service/api/json_v2.php/CN_MarketData.getKLineData?symbol=sh600000&scale=240&ma=no&datalen=100
```

| 参数 | 说明 |
|------|------|
| symbol | 股票代码（sh/sz前缀） |
| scale | K线周期：5/15/30/60/240（分钟），240=日线 |
| ma | 均线：no=不包含 |
| datalen | 返回条数 |

**返回字段：** day, open, high, low, close, volume

**注意：** 调用时需设置 `Referer: https://finance.sina.com.cn`

## 5. 数据架构方案

### 5.1 分层架构

```
┌─────────────────────────────────────────────────────────────┐
│                     选股 Workflow 数据架构                    │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌──────────────────────────────────┐                       │
│   │        AKShare（主力数据源）        │  ← 日常所有数据获取    │
│   │   覆盖: 行情 + 财务 + 资金         │                       │
│   │         + 板块 + 宏观              │                       │
│   └──────────────┬───────────────────┘                       │
│                  │                                           │
│                  ▼  盘后全量交叉校验                             │
│   ┌──────────────────────────────────┐                       │
│   │     Tushare（交叉验证数据源）       │  ← 财务数据 + 估值指标  │
│   │   重点: 利润表 / 负债表 / PE/PB    │     每日收盘后批量校验    │
│   │         / ROE / 净利润            │     而非实时逐条校验      │
│   └──────────────────────────────────┘                       │
│                                                             │
│   ┌──────────────────────────────────┐                       │
│   │     Baostock（备用 #1）            │  ← 历史K线兜底          │
│   └──────────────────────────────────┘                       │
│   ┌──────────────────────────────────┐                       │
│   │     efinance（备用 #2）            │  ← 行情 + 财务兜底      │
│   └──────────────────────────────────┘                       │
│   ┌──────────────────────────────────┐                       │
│   │     腾讯接口（备用 #3）             │  ← 实时行情兜底         │
│   └──────────────────────────────────┘                       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 各角色职责

| 角色 | API | 职责 | 触发时机 | 调用量 |
|------|-----|------|---------|-------|
| **主力** | AKShare | 所有日常数据获取 | 每次请求 | 高频 |
| **验证** | Tushare | 对关键指标做偏差校验 | 每日收盘后批量 | 低频 |
| **备用1** | Baostock | AKShare + Tushare 都挂时，提供历史K线 | 故障切换 | 极低 |
| **备用2** | efinance | 上述都挂时，提供行情 + 基础财务 | 故障切换 | 极低 |
| **备用3** | 腾讯接口 | 只需实时行情时极简获取 | 故障切换 | 极低 |

### 5.3 交叉验证策略

#### 为什么选 AKShare + Tushare？

| 方案 | 数据源独立性 | 验证覆盖面 | 推荐度 |
|------|------------|-----------|-------|
| AKShare + Baostock | ✅ 最强（独立数据库） | ⚠️ 偏弱（仅K线+财务报表） | ⭐⭐⭐ |
| **AKShare + Tushare** | ✅ **强（独立数据体系）** | ✅ **最广（行情+财务+估值+资金）** | ⭐⭐⭐⭐⭐ |
| efinance + AKShare | ⚠️ 弱（同源东方财富） | ✅ 广 | ⭐⭐ |

#### 验证执行方式

```
┌─ 盘中实时（用户无感，毫秒级）────────────────────────┐
│  用户请求 → AKShare → 返回结果                        │
└──────────────────────────────────────────────────────┘

┌─ 盘后批量（每天收盘后，后台跑）────────────────────────┐
│  从 AKShare 获取所有关键指标                             │
│  从 Tushare 获取同样指标                                │
│  逐字段对比 → 标记偏差 → 写入可信数据表                   │
└──────────────────────────────────────────────────────┘

┌─ 盘中增量（每隔30分钟，后台更新）───────────────────────┐
│  增量拉取 + 对比 → 更新缓存表                             │
└──────────────────────────────────────────────────────┘
```

#### 关键验证指标清单

| 类别 | 验证字段 | 重要性 | 偏差阈值 |
|------|---------|-------|---------|
| 行情 | 收盘价 | 🔴 关键 | > 0.01元 |
| 行情 | 成交量 | 🟡 中等 | > 5% |
| 估值 | PE（市盈率） | 🔴 关键 | > 5% |
| 估值 | PB（市净率） | 🔴 关键 | > 5% |
| 财务 | 净利润 | 🔴 关键 | > 3% |
| 财务 | 营业收入 | 🔴 关键 | > 3% |
| 财务 | ROE | 🟡 中等 | > 5% |
| 财务 | 毛利率 | 🟡 中等 | > 5% |

#### 校验投票逻辑

```
从 AKShare、Tushare、Baostock、efinance、腾讯接口 获取同一指标
    ↓
过滤掉无数据的 API
    ↓
剩余值 ≥ 3 个？
    ├─ 是 → 取中位数，检测最大偏差
    │        ├─ 偏差 ≤ 5% → 标记为「高可信」
    │        └─ 偏差 > 5%  → 标记为「有偏差」，记录日志
    └─ 否 → 信任 AKShare，标记为「低可信」
```

## 6. 风险与注意事项

### 6.1 免费接口通用风险

| 风险 | 说明 | 应对策略 |
|------|------|---------|
| **接口变更/下线** | 非官方接口可能随时变更或停止服务 | 多数据源冗余 + 故障切换 |
| **IP 频率限制** | 短时间高频请求会被封禁 | 控制请求频率 + 代理池 |
| **数据延迟** | 免费数据可能有 15-30 分钟延迟 | 接受延迟，标记数据时间戳 |
| **数据缺失** | 某些 API 对特定字段返回空值 | 多源对比，自动补全 |
| **Referer 限制** | 新浪/腾讯需设置特定 Referer 头 | 请求中统一添加 |

### 6.2 各数据源特有注意

| 数据源 | 特有风险 |
|--------|---------|
| AKShare | 接口多来自爬虫，上游网站改版可能导致失效 |
| Baostock | 无实时行情，仅适合历史数据 |
| Tushare | 积分制限制，高级接口需攒积分；曾经历停运 |
| efinance | 底层依赖东方财富，与 AKShare 存在同源风险 |
| AData | 覆盖面窄，仅适合A股行情 |
| 新浪财经 | 需设置 Referer，否则可能被拒绝 |
