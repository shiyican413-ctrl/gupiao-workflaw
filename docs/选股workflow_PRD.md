# 选股 Workflow 项目 PRD

## 1. 项目概述

### 1.1 项目名称

选股 Workflow

### 1.2 产品定位

面向个人投资者、量化初学者和投研辅助场景，提供一套从“选股条件配置 -> 数据筛选 -> 多维评分 -> 候选股票池 -> 跟踪复盘”的半自动化工作流工具。

产品不直接给出买卖建议，不承诺收益，不替代专业投资顾问服务。核心价值是帮助用户更高效、可复盘、可解释地构建自己的股票观察池。

### 1.3 背景与问题

个人投资者在选股时常见问题：

- 数据分散，需要在行情软件、财报网站、新闻平台之间反复切换。
- 筛选逻辑不稳定，容易受短期情绪和热点影响。
- 缺少统一评分体系，难以比较不同股票。
- 缺少复盘机制，不知道选股条件是否长期有效。
- 手工维护观察池成本高，容易遗漏风险事件。

本项目希望把选股过程产品化、流程化、结构化。

## 2. 产品目标

### 2.1 核心目标

- 建立一套可配置的选股工作流。
- 支持基本面、技术面、资金面、估值、成长性、风险等多维筛选。
- 为每只股票生成可解释的评分和入选原因。
- 帮助用户沉淀自己的股票池和复盘记录。
- 降低手工整理数据的成本。

### 2.2 非目标

- 不提供自动实盘交易。
- 不承诺推荐股票未来上涨。
- 不提供个性化投资顾问服务。
- MVP 阶段不做复杂高频量化交易。
- MVP 阶段不做完整券商账户接入。

## 3. 目标用户

### 3.1 核心用户

个人投资者：

- 有一定股票基础。
- 希望用规则筛选股票，而不是完全依赖消息面。
- 关注 A 股、美股或港股中的一个或多个市场。
- 愿意定期复盘自己的选股逻辑。

### 3.2 次级用户

量化初学者：

- 希望从可视化规则开始搭建简单策略。
- 需要理解因子、打分、回测的基本过程。

投研辅助用户：

- 需要快速生成候选股票池。
- 需要导出筛选结果，用于进一步分析。

## 4. 用户核心场景

### 4.1 快速筛选股票池

用户希望在几分钟内得到一批符合条件的候选股票，例如：

- 市值大于 100 亿。
- 近 3 年营收增长为正。
- ROE 高于行业中位数。
- PE 不高于过去 5 年分位数 70%。
- 近 20 日趋势向上。

系统输出候选股票列表、综合评分、入选原因和风险提示。

### 4.2 创建自定义选股策略

用户希望保存自己的筛选模板，例如：

- 低估值高分红策略。
- 高成长策略。
- 趋势突破策略。
- 业绩反转策略。
- 行业龙头策略。

系统允许用户配置条件、权重、市场范围、行业范围，并保存为 workflow。

### 4.3 每日更新观察池

用户每天打开系统，看到：

- 新进入候选池的股票。
- 被移出候选池的股票。
- 评分上升或下降明显的股票。
- 出现财报、公告、异动、舆情风险的股票。

### 4.4 复盘策略有效性

用户希望知道过去一段时间自己的选股逻辑是否有效：

- 入选股票后 5 日、20 日、60 日表现。
- 与沪深 300、标普 500、恒生指数等基准比较。
- 胜率、平均收益、最大回撤。
- 哪些因子贡献更高。

## 5. 核心流程

```text
选择市场/行业
    -> 选择或创建选股 workflow
    -> 配置筛选条件和评分权重
    -> 拉取并清洗数据
    -> 初筛股票池
    -> 多因子评分
    -> 生成候选股票列表
    -> 展示入选原因与风险提示
    -> 加入观察池
    -> 定期更新和复盘
```

## 6. 功能需求

### 6.1 市场与股票范围

MVP 支持：

- A 股基础股票池。
- 按行业、市值、上市时间、交易状态过滤。
- 剔除 ST、退市整理、长期停牌股票。

后续版本支持：

- 美股。
- 港股。
- ETF。
- 可转债。

### 6.2 选股 Workflow 管理

用户可以：

- 创建 workflow。
- 编辑 workflow 名称、描述、适用市场。
- 设置筛选条件。
- 设置评分因子和权重。
- 复制已有 workflow。
- 启用、停用、删除 workflow。
- 查看 workflow 历史运行记录。

内置模板：

- 价值低估型。
- 成长质量型。
- 高股息型。
- 趋势动量型。
- 财报改善型。
- 行业龙头型。

### 6.3 筛选条件模块

基础条件：

- 市场。
- 行业。
- 总市值。
- 流通市值。
- 上市年限。
- 是否 ST。
- 是否停牌。

基本面条件：

- 营业收入增长率。
- 净利润增长率。
- 毛利率。
- 净利率。
- ROE。
- ROA。
- 资产负债率。
- 经营现金流。

估值条件：

- PE。
- PB。
- PS。
- 股息率。
- PEG。
- 估值历史分位数。
- 行业估值分位数。

技术面条件：

- 5 日、20 日、60 日均线状态。
- 近 N 日涨跌幅。
- 成交量变化。
- 波动率。
- 突破近期高点。
- 距离近期低点的幅度。

资金面条件：

- 主力资金净流入。
- 北向资金持仓变化。
- 换手率。
- 成交额。

风险条件：

- 财务造假风险标签。
- 大股东减持。
- 质押比例。
- 重大诉讼。
- 监管处罚。
- 业绩预告大幅下修。

### 6.4 多因子评分

系统为每只股票生成 0-100 分综合评分。

默认评分维度：

- 成长性：营收增长、利润增长、现金流增长。
- 盈利质量：ROE、毛利率、净利率、经营现金流。
- 估值吸引力：PE、PB、PEG、历史估值分位。
- 趋势强度：均线结构、动量、成交量。
- 资金关注度：资金流入、换手活跃度。
- 风险扣分：财务异常、监管风险、业绩恶化、流动性不足。

示例公式：

```text
综合评分 =
成长性评分 * 成长权重
+ 盈利质量评分 * 质量权重
+ 估值评分 * 估值权重
+ 趋势评分 * 趋势权重
+ 资金评分 * 资金权重
- 风险扣分
```

要求：

- 每个评分必须可解释。
- 展示每个维度的分数和排名。
- 展示该股票入选的主要原因。
- 展示主要风险项。

### 6.5 结果页

结果列表字段：

- 股票代码。
- 股票名称。
- 行业。
- 当前价格。
- 涨跌幅。
- 总市值。
- 综合评分。
- 成长评分。
- 估值评分。
- 趋势评分。
- 风险等级。
- 入选原因。
- 更新时间。

支持操作：

- 排序。
- 筛选。
- 搜索。
- 查看详情。
- 加入观察池。
- 导出 CSV / Excel。
- 对比多只股票。

### 6.6 股票详情页

详情页包含：

- 基础信息。
- 价格走势。
- 财务摘要。
- 估值区间。
- 行业对比。
- 因子评分雷达图。
- 入选原因。
- 风险提示。
- 历史评分变化。
- 相关公告和新闻摘要。

### 6.7 观察池

用户可以：

- 将股票加入观察池。
- 设置标签。
- 添加备注。
- 查看加入时评分。
- 查看当前评分。
- 查看评分变化。
- 设置提醒条件。
- 移出观察池。

提醒示例：

- 综合评分跌破 70。
- 股价跌破 20 日均线。
- 估值分位超过 80%。
- 发布财报。
- 出现重大公告。

### 6.8 复盘与回测

MVP 支持轻量复盘：

- 记录每次 workflow 运行结果。
- 查看入选股票后 5 日、20 日、60 日涨跌幅。
- 与基准指数对比。
- 查看命中率。
- 查看平均收益。

后续版本支持：

- 完整历史回测。
- 调仓频率配置。
- 交易成本模拟。
- 最大回撤。
- 夏普比率。
- 因子贡献分析。

## 10. 风险与合规

必须明确展示：

- 本产品仅作为数据分析和投研辅助工具。
- 不构成任何投资建议。
- 股票投资有风险，历史表现不代表未来收益。
- 用户应独立判断并自行承担投资风险。

产品设计要求：

- 避免使用“稳赚”“必涨”“买入信号”等承诺性表达。
- 推荐用“候选股票”“观察池”“入选原因”“风险提示”等中性表达。
- 高风险股票必须展示明显风险标签。
- 所有评分均需展示计算逻辑或解释口径。

## 11. MVP 范围

### 11.1 MVP 必做

- A 股股票池。
- 基础行情数据接入。
- 财务指标接入。
- 估值指标接入。
- 创建和编辑 workflow。

- 观察池。

### 11.2 MVP 不做

- 自动交易。
- 券商账户接入。
- 高频分钟级策略。
- 复杂组合优化。
- 多人协作。

- 付费订阅系统。

## 12. 成功指标

产品使用指标：

- 用户创建 workflow 数量。
- 每周运行 workflow 次数。
- 观察池股票数量。
- 导出结果次数。
- 复盘页访问次数。

结果质量指标：

- 入选股票后 20 日相对基准表现。
- workflow 胜率。
- 用户保留率。
- 用户主动调整 workflow 的频率。

体验指标：

- 从创建 workflow 到生成结果不超过 3 分钟。
- 常规 workflow 运行时间不超过 30 秒。
- 结果页首屏加载不超过 2 秒。

## 13. 技术建议

### 13.1 前端

建议：

- Vue 3。
- Vite。
- TypeScript。
- UI 组件库使用 Element Plus，优先满足后台工作台、表格、表单、弹窗、筛选器等场景。
- 状态管理使用 Pinia。
- 路由使用 Vue Router。
- 表格组件支持排序、筛选、固定列、虚拟滚动和导出。
- 图表使用 ECharts。
- UI 风格以投研工具为主，信息密度高、克制、清晰。

### 13.2 后端

建议：

- Python FastAPI。
- 定时任务使用 APScheduler / Celery。
- 数据处理使用 Pandas。
- 指标计算模块独立封装。

### 13.3 数据库

MVP：

- SQLite 或 PostgreSQL。

正式版本：

- PostgreSQL。
- Redis 缓存。
- 对历史行情可考虑 ClickHouse。

- MVP 是本地桌面工具、Web 应用，还是云端 SaaS？
- 是否需要中文自然语言生成选股解释？
- 是否需要接入大模型做公告和新闻摘要？

## 18. 推荐 MVP 版本定义

建议第一版聚焦为：

一个本地或私有部署的 A 股选股工作台。用户可以创建选股 workflow，配置基础面、估值和趋势条件，运行后生成候选股票池，并将股票加入观察池。系统每天更新数据，记录每次运行结果，并提供简单复盘。

这个范围足够小，可以较快做出可用产品；同时又保留了后续扩展到回测、AI 摘要、多市场和自动化提醒的空间。
